8// Typed actions queued onto MinecraftServer from outside the server
9// thread (UI, network, save manager). Each variant alternative is a
10// plain data struct describing the requested operation; the server
11// drains the queue in its tick loop and dispatches via std::visit.
12//
13// This replaces the old eXuiServerAction enum + per-pad polling +
14// XuiActionPayload variant. The previous design forced the server to
15// poll IGameServices every tick and pull a UI-shaped payload through a
16// polymorphic base; now the server owns its own queue and the action
17// types live alongside the consumer.
